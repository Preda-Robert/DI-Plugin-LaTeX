\chapter*{Project summary}
\addcontentsline{toc}{chapter}{Project summary}
\label{summary}

This project implements a \textbf{Visual Studio Code extension} for C\# projects using \texttt{Microsoft.Extensions.DependencyInjection}. The extension parses constructor signatures with \texttt{tree-sitter}, scans registration patterns across the workspace, and reports DI issues through editor diagnostics and command-palette actions. Developers can obtain suggested \texttt{services.Add*} lines, generated factory classes for services with many interface dependencies, and automated insertion into \texttt{CompositionRoot} or \texttt{AddApplicationServices} methods.

The architecture separates a language-agnostic analyzer (\texttt{analyzer.ts}) from the VS Code host (\texttt{extension.ts}), with auxiliary modules for composition-root edits and interface extraction. The design prioritizes fast feedback and copy-pasteable output over full semantic completeness. Sample applications in the repository validate registration, factory generation, and merge workflows.

\textbf{Keywords:} dependency injection, Visual Studio Code extension, C\# static analysis, tree-sitter, \texttt{Microsoft.Extensions.DependencyInjection}, software architecture, code generation.
