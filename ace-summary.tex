\chapter*{Project summary}
\addcontentsline{toc}{chapter}{Project summary}
\label{summary}

This project implements a \textbf{Visual Studio Code extension} for C\# projects using \texttt{Microsoft.Extensions.DependencyInjection}. The extension parses constructor signatures with \texttt{tree-sitter}, scans registration patterns across the workspace (including \texttt{AddDbContext}, \texttt{AddAutoMapper}, and ASP.NET Identity), and reports DI issues through editor diagnostics and Command Palette actions. Developers obtain suggested \texttt{services.Add*} lines with lifetimes inferred from the composition root, generated factory classes for services with three or more interface dependencies, and automated insertion or merge into \texttt{CompositionRoot} or \texttt{AddApplicationServices} methods.

The architecture separates a language-agnostic analyzer (\texttt{analyzer.ts}) from the VS Code host (\texttt{extension.ts}), with auxiliary modules for composition-root edits and interface extraction. Six sample workspaces (\texttt{BaseSample}, \texttt{BuilderFactorySample}, \texttt{CheckoutSample}, \texttt{MultiFactorySample}, \texttt{MealPlanner}, \texttt{EventPlanner}) support manual validation documented in the thesis evaluation chapter, including regression cases for 32\,KiB parse limits and framework false positives.

\textbf{Keywords:} dependency injection, Visual Studio Code extension, C\# static analysis, tree-sitter, \texttt{Microsoft.Extensions.DependencyInjection}, software architecture, code generation, empirical validation.
