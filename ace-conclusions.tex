\section{Summary of contributions}

This work presented a VS Code extension that assists C\# developers with \texttt{Microsoft.Extensions.DependencyInjection}. The solution combines a tree-sitter--based analyzer module with a heuristic workspace aggregator hosted in the Extension API. Together they provide constructor dependency insight, DI smell detection, registration suggestions, optional factory code generation, and composition-root insertion workflows.

The modular split---\texttt{analyzer.ts} for syntax facts, \texttt{extension.ts} for orchestration, \texttt{registrationInsert.ts} and \texttt{extractInterface.ts} for targeted edits---keeps responsibilities testable and documents a practical architecture for small language-aware IDE tools.

\section{Evaluation against objectives}

The sample workspaces (\texttt{BuilderFactorySample}, \texttt{CheckoutSample}, \texttt{MultiFactorySample}, \texttt{MealPlanner}) demonstrate that:
\begin{itemize}
	\item factory generation activates for services with three or more interface dependencies and produces usable \texttt{*Factory.cs} templates;
	\item registration suggestions align with composition-root lifetime conventions when a root file exists;
	\item false positives for \texttt{IMapper}, \texttt{IServiceProvider}, and Identity managers were reduced by extended registration detection;
	\item separating factory output from \texttt{services.Add*} suggestions avoids redundant \texttt{AppBuilder} content in the factory command.
\end{itemize}

\section{Limitations}

The approach remains \textbf{heuristic and syntactic}, not a substitute for Roslyn-powered refactorings or runtime DI validation. Large C\# files exceed the tree-sitter buffer limit; cross-file concrete-type smells may persist; and empty extracted interfaces require manual completion.

\section{Future work}

Recommended extensions include: Roslyn integration or LSP collaboration for symbol-accurate analysis; configurable factory thresholds; support for additional registration APIs; undo-friendly refactorings; unit tests over golden-file samples; and optional export of dependency graphs (e.g.\ Graphviz) for architecture reviews.
